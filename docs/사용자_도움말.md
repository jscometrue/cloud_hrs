# JSCORP HR 시스템 사용자 도움말

**버전**: 1.0  
**대상**: 인사시스템 사용자(직원, 관리자, HR 담당자)

---

## 1. 시작하기

### 1.1 로그인

1. **사용자 ID**와 **비밀번호**를 입력한 뒤 **로그인**을 클릭합니다.
2. 처음 사용 시 관리자(ADMIN)에게 계정 생성을 요청하거나, 비밀번호 초기화가 필요하면 **비밀번호 찾기**에서 사용자 ID를 입력 후 **비밀번호 초기화 요청**을 누릅니다. 화면에 표시된 링크로 접속해 새 비밀번호를 설정할 수 있습니다.
3. 백엔드에 연결되지 않을 때는 로그인 화면의 **API 주소 설정**에서 서버 주소를 입력한 뒤 **저장 후 새로고침**을 실행합니다.

### 1.2 화면 구성

- **상단**: JSCORP HR 제목, 언어 선택, (권한에 따라) 권한 요청 버튼, 비밀번호 변경, 로그아웃
- **탭 메뉴**: 대시보드, 직원, 조직, 근태, 급여, 평가, 교육, 복리후생, (관리자만) 사용자
- **본문**: 선택한 탭에 따른 목록·입력·조회 화면

### 1.3 권한(역할) 안내

| 역할 | 설명 | 볼 수 있는 메뉴/데이터 |
|------|------|------------------------|
| **EMPLOYEE** | 일반 직원 | 대시보드, 본인 직원 정보, 본인 근태·급여·평가·교육·복리후생 |
| **MANAGER** | 팀/부서 관리자 | 직원(같은 부서), 근태·휴가 승인(부서), 팀 평가 입력 |
| **HR_ADMIN** | 인사 관리자 | 직원·조직·근태·급여·평가·교육·복리후생 전반, 사용자·권한 요청 승인 |
| **ADMIN** | 시스템 관리자 | 위와 동일 + 사용자 추가/비밀번호 재설정, 승진 후보 등 |

## 2. 메뉴별 사용 방법

### 2.1 대시보드

- **전체 직원**: 등록된 직원 수  
- **재직**: 상태가 재직인 직원 수  
- **부서 수**, **급여그룹 수**, **급여 차수**: 조직·급여 기본 현황  
- **휴가 대기**: 승인 대기 중인 휴가 신청 건수  
- **이직률**: (전체−재직)/전체 비율(%)  
- **총 급여**: 급여 결과 합계(백만 단위 표시)

---

### 2.2 직원

- **목록**: 권한에 따라 본인만, 같은 부서만, 또는 전체 직원이 보입니다.
- **직원 추가**: **직원 추가** 버튼 → 사번, 이름, 성, 이메일, 연락처, 입사일, 상태, 부서, 급여그룹 입력 후 저장.  
  - 사번과 동일한 로그인 계정이 없으면 자동 생성됩니다.
- **수정**: 해당 직원 행의 **수정** → 항목 변경 후 저장. 부서/상태 변경 시 이력이 자동 기록됩니다.
- **삭제**: **삭제** 클릭 후 확인. 삭제 시 연결된 로그인 계정은 비활성 처리됩니다.

---

### 2.3 조직

- **부서**: 부서 목록 조회, **부서 추가**로 코드·명칭 입력, **수정**/**삭제**로 변경·삭제.
- **급여그룹**: 급여그룹 목록, **급여그룹 추가**로 코드·명칭·급여 주기·마감일·지급일 설정.
- **급여항목**: 급여항목(코드, 명칭, 항목 유형, 과세, 산정 방식, 기본 금액) 추가·수정·삭제.

---

### 2.4 근태

- **근무형태**: 근무유형(코드, 명칭, 시작/종료 시각, 휴게 분) 등록·수정·삭제.
- **휴가 신청**: **휴가 신청** 버튼 → 대상 직원, 휴가 유형, 시작/종료 일시, 시간, 사유 입력 후 **신청**.  
  - MANAGER/HR_ADMIN/ADMIN은 목록에서 **승인**/**반려** 처리할 수 있습니다.
- **월별 집계**: 선택한 년월의 근태 요약(근무 시간, 초과근무, 지각·조퇴·결근 등)을 조회합니다.
- **출퇴근 기록**: **출근/퇴근 기록** 버튼으로 현재 시각 기준 IN/OUT 기록. 아래 목록에서 해당 월의 출퇴근 기록을 확인할 수 있습니다.

---

### 2.5 급여

- **급여 차수**: 차수(기간·유형) 목록에서 한 건을 선택하면 해당 차수의 **급여 결과**(직원별 총지급·공제·실지급)가 표시됩니다.
- **급여 차수 생성**: **급여 차수 생성** → 급여그룹, 년월(YYYYMM), 유형(정기/상여/퇴직) 입력 후 **생성**.
- HR_ADMIN/ADMIN은 차수 선택 후 **급여 계산** API를 호출해 해당 차수의 급여를 산정할 수 있습니다(실제 계산 로직은 환경에 따라 다름).

---

### 2.6 평가

- **내 평가**  
  - **평가 계획** 드롭다운에서 계획 선택.  
  - 해당 계획에 **평가 항목**이 있으면: 항목별 **점수**(0~100)와 **코멘트** 입력 후 **저장**.  
  - 평가 항목이 없으면: **평가 점수(0~100)**와 **코멘트**만 입력 후 저장.
- **팀 평가**(MANAGER/HR_ADMIN/ADMIN)  
  - **평가 대상**에서 평가할 직원 선택.  
  - 항목별 점수·코멘트 입력 후 **저장**하면 해당 직원에 대한 상사/다면 평가로 저장됩니다.
- **평가 계획 목록**(HR_ADMIN/ADMIN): 등록된 평가 계획(연도·명칭·상태·기간) 조회.
- **승진 후보**(HR_ADMIN/ADMIN): 선택한 계획에 대해 등급 정책에서 “승진 후보”로 설정된 구간에 해당하는 직원 목록 조회.  
  - 먼저 **등급 정책**을 설정하고 **집계**를 실행한 뒤 확인할 수 있습니다.

*관리자용: 평가 항목·등급 정책·대상 시드·집계는 API로만 제공됩니다. 필요 시 개발/운영 담당자에게 요청하세요.*

---

### 2.7 교육

- **교육 과정**: 등록된 과정(코드·명칭·카테고리) 목록 조회.
- **교육 회차**: 회차(제목·과정·기간) 목록에서 **수강 신청** 버튼으로 신청.
- **내 수강 내역**: 본인이 신청한 회차와 상태(신청/승인/이수/취소) 조회.  
  - 이수·승인·취소 처리는 HR_ADMIN/ADMIN이 API로 수행합니다.

---

### 2.8 복리후생

- **내 포인트 잔액**: 연도별·정책별 포인트 잔액 조회.
- **복지 정책**: 사용 가능한 복지 정책(코드·명칭·유형·기본 포인트) 목록 조회.  
  - 잔액 생성·포인트 적립/사용은 HR_ADMIN/ADMIN이 API로 처리합니다.

---

### 2.9 사용자 (ADMIN / HR_ADMIN 전용)

- **사용자 목록**: 로그인 계정(사용자 ID, 활성 여부, 역할) 조회.  
  - **비밀번호 재설정**: 해당 사용자 행에서 버튼 클릭 후 새 비밀번호 입력.
- **신규 사용자**: **신규 사용자** 버튼 → 사용자 ID, 비밀번호, 역할(EMPLOYEE 등) 입력 후 추가.
- **권한 요청 승인**: 다른 사용자가 요청한 권한(역할) 상승을 **승인** 또는 **반려**합니다.

**권한 요청**(일반 사용자): 상단 **권한 요청** 버튼 → 요청할 역할, 사유 입력 후 **요청 보내기**. ADMIN/HR_ADMIN이 승인하면 해당 역할로 변경됩니다.

---

## 3. 공통 기능

- **비밀번호 변경**: 로그인 후 상단 **비밀번호 변경** → 현재 비밀번호, 새 비밀번호 입력 후 저장.
- **언어**: 상단에서 한국어/영어 전환 가능(화면 라벨만 변경).
- **오류 메시지**: 작업 실패 시 본문 상단에 빨간색으로 메시지가 표시됩니다. 내용을 확인한 뒤 재시도하거나 관리자에게 문의하세요.

## 4. 구현된 기능 요약

| 영역 | 구현 내용 |
|------|-----------|
| **조직·인사** | 부서·급여그룹·급여항목 CRUD, 직원 CRUD 및 부서/상태 변경 이력 |
| **근태** | 근무형태, 휴가 신청·승인·반려, 월별 근태 집계, 근태 마감, 출퇴근 기록 |
| **급여** | 급여 차수·결과 조회, 급여 계산 실행 |
| **평가** | 평가 계획, 평가 항목·가중치, 자기평가(단일/항목별), 팀(상사·다면) 평가, 등급 정책·집계·승진 후보 |
| **교육** | 교육 과정·회차 조회, 수강 신청, 내 수강 내역 |
| **복리후생** | 복지 정책, 포인트 잔액·거래(API), 내 잔액 조회 |
| **시스템** | 사용자 관리, 역할별 권한, 권한 요청·승인, 공통코드·감사 로그(API), 대시보드 KPI(이직률·총 급여) |

## 5. 문의 및 문제 해결

- **로그인이 안 될 때**: 사용자 ID·비밀번호 확인, 비밀번호 초기화 링크 사용, API 주소 설정 확인.
- **데이터가 안 보일 때**: 본인 역할에서 조회 가능한 범위인지 확인(직원은 본인/부서/전체 등).
- **특정 버튼·메뉴가 없을 때**: 해당 기능은 역할에 따라 숨겨질 수 있습니다. 필요 시 권한 요청을 하거나 관리자에게 문의하세요.
- **기술 문의**: API 상세, 배치·연동, 감사 로그 조회 등은 시스템 관리자 또는 개발 담당자에게 요청하세요.
