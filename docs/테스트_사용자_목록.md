# JSCORP HR 테스트 사용자 목록

시스템 기동 시 시드 데이터로 생성·갱신되는 테스트 계정입니다.  
**비밀번호는 개발/테스트용이므로 운영 환경에서는 반드시 변경하세요.**

---

## 테스트 계정 (ID / 비밀번호)

| 역할 | 사용자 ID | 비밀번호 | 비고 |
|------|-----------|----------|------|
| **ADMIN** (시스템 관리자) | `admin` | `admin123` | 전체 메뉴, 사용자 관리, 권한 승인 |
| **HR_ADMIN** (인사 관리자) | `hradmin` | `hr123` | 인사·근태·급여·평가·교육·복리후생 전반, 권한 승인 |
| **MANAGER** (팀/부서 관리자) | `manager1` | `mgr123` | 같은 부서 직원 조회, 휴가 승인, 팀 평가 입력 |
| **EMPLOYEE** (일반 직원) | `sample1` | `sample123` | 본인 데이터만 조회·입력 |
| **EMPLOYEE** | `sample2` | `sample123` | 본인 데이터만 조회·입력 |
| **EMPLOYEE** | `sample3` | `sample123` | 본인 데이터만 조회·입력 |

---

## 요약

- **관리자 로그인**: `admin` / `admin123`
- **인사 담당자**: `hradmin` / `hr123`
- **팀장**: `manager1` / `mgr123`
- **일반 직원**: `sample1`~`sample3` / `sample123`

직원(sample1~3, manager1)이 로그인해 “내” 데이터를 보려면, 해당 사용자와 **직원(Employee)** 레코드가 `user_id`로 연결되어 있어야 합니다.  
직원 등록 시 로그인 ID를 사번(emp_no)과 맞추거나, 사용자 관리에서 직원과 계정을 연결해 두면 됩니다.
